## 二、功能规划（Superpowers思考）

### 2.1 核心指标卡片（新增）
**思考维度**: 用户打开概览页第一眼希望看到什么？
- 今日/本周/本月关键指标快速概览
- 与上一周期的对比变化
- 指标：总销售额、总利润、订单数量、平均利润率

**实现方案**:
- 新增4个核心指标卡片，位于页面顶部
- 每个卡片显示：指标名称、当前值、环比变化（箭头+百分比）
- 支持点击卡片下钻查看详情

### 2.2 数据可视化增强
**思考维度**: 如何让数据更直观、更有洞察力？

#### 2.2.1 销售额占比饼图
- 按产品分类统计销售额占比
- 使用环形图展示，中心显示总额
- 支持图例点击筛选

#### 2.2.2 客户价值分布图
- 客户分层分析（高价值/中价值/低价值客户分布）
- 使用柱状图或散点图

#### 2.2.3 时间维度对比
- 同比分析（与去年同期对比）
- 环比分析（与上月/上周对比）
- 使用双折线图展示

### 2.3 智能分析功能
**思考维度**: 如何让系统主动提供业务洞察？

#### 2.3.1 异常检测
- 自动识别异常订单（利润过低、亏损订单）
- 数据波动预警（销售额突降/突增）

#### 2.3.2 智能建议
- 基于数据生成业务建议
- 例如："产品X近7天销量下降30%，建议关注库存"

### 2.4 交互体验优化
**思考维度**: 如何让用户更高效地使用？

#### 2.4.1 快捷筛选
- 预设时间范围快捷按钮（今日、本周、本月、本年）
- 自定义日期范围选择器优化

#### 2.4.2 数据刷新
- 手动刷新按钮
- 自动刷新开关（5分钟/10分钟/30分钟间隔）
- 最后更新时间显示

#### 2.4.3 数据导出
- 支持导出当前筛选条件下的所有数据
- 格式：Excel、PDF、CSV

### 2.5 响应式与性能优化
**思考维度**: 不同设备、大数据量下的体验

- 移动端适配优化（当前布局在移动端可能过于拥挤）
- 大数据量分页加载
- 图表懒加载/虚拟滚动
- 数据缓存机制（Redis）

## 四、技术方案

### 4.1 新增API列表

| API路径 | 方法 | 功能 | 优先级 |
|--------|------|------|--------|
| `/overview/core-metrics` | GET | 获取核心指标数据 | P1 |
| `/overview/sales-distribution` | GET | 销售额分布统计 | P2 |
| `/overview/profit/statistics` | GET | 扩展支持同比/环比 | P2 |
| `/overview/export` | POST | 数据导出 | P3 |
| `/overview/anomalies` | GET | 异常数据检测 | P3 |

### 4.2 新增组件列表

| 组件名称 | 路径 | 功能 | 优先级 |
|---------|------|------|--------|
| `CoreMetrics` | `overview/components/CoreMetrics.tsx` | 核心指标卡片 | P1 |
| `SalesPieChart` | `overview/components/SalesPieChart.tsx` | 销售额占比饼图 | P2 |
| `RefreshControl` | `overview/components/RefreshControl.tsx` | 刷新控制栏 | P1 |
| `QuickDateFilter` | `overview/components/QuickDateFilter.tsx` | 快捷日期筛选 | P1 |
| `AnomalyAlert` | `overview/components/AnomalyAlert.tsx` | 异常警示栏 | P3 |

### 4.3 数据库变更

无需变更现有表结构，所有统计基于现有 `t_order` 表进行聚合查询。

如需优化性能，可考虑：
- 添加联合索引：`(createUser, orderTime)`
- 添加覆盖索引：`(createUser, orderTime, name, buyPrice, sellPrice, number, otherCost)`

### 4.4 缓存策略

```
Redis Key 设计:
- overview:core:{userId}:{dateRange} -> 核心指标数据 (TTL: 5min)
- overview:top5:{userId}:{dateRange}:{type} -> TOP5数据 (TTL: 5min)
- overview:profit:{userId}:{days} -> 利润趋势数据 (TTL: 5min)
```
